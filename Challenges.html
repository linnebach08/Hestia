Challenges

Expected Hardware and Software Challenges for the Project

Software Challenges

  • Configuring a reliable communication protocol between the app and sensors
    • Current Plan: Use bluetooth to communicate between the app and sensors. This has been proven to be reliable in past projects.
  • Designing the database which will form the backend of our app
    • Current Plan: We will choose a reliable and cheap SQL database engine. Then, we will create our database server and structure it around the entity-relationship model.
  • Making the app look visually appealing to users
    • Current Plan: We will beta test the app design with users before release and two of our members already have app design experience.

Hardware Challenges

  • Making the bracelet heart rate monitor small enough to fit comfortably on the wrist and accurately measure the heart rate
    • Current Plan: We know it has been done before so it is possible. The components should be small enough to make it work and we are all currently getting embedded system design experience.
  • Figuring out battery/charging for the bracelet and straw
    • Current Plan: We can either have our systems run on solar power or we can make our bracelet and straw rechargable through a standard USB-C cable.
  • Configuring the liquid sensor to accurately measure water intake
    • Current Plan: We have found a water sensor that we can use to measure the water depth and then create an embedded system to read and transmit this data. Then we can use depth to measure volume by using a water bottle with a known size.
